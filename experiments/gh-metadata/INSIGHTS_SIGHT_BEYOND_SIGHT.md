# Phalanx Duel: Sight Beyond Sight (DevEx Forensics)

This document moves beyond traditional DORA metrics into **High-Fidelity Context Analysis**. It leverages the GitHub Metadata Harvester to identify the invisible forces shaping the project estate.

## 1. The Ghost Areas (Knowledge Decay)
*Metric: Days since last active human contribution per functional label.*

**Sight Beyond Sight**: Identifies codebases that are functional but "Contextually Orphaned." When the primary holder leaves, these areas become "Knowledge Voids" that require high-energy extraction to recover.

## 2. Agile Lifecycle Funnel (Linger Time)
*Metric: Median duration for Issue -> PR -> Merge.*

- **Spec-to-Code**: Time spent in the "Intent" phase. High latency here suggests a bottleneck in requirements or prioritization.
- **Code-to-Merge**: Time spent in the "Execution" phase. High latency here suggests "Feedback Loop Friction" or slow CI environments.
- **Issue Linger**: Identifies "Stalled Intent"—issues that have been open for months without a PR link, indicating a decaying backlog.

## 3. Defensive Gravity (Test Resonance)
*Metric: Ratio of test churn to functional churn.*

**Sight Beyond Sight**: Healthy repositories exhibit a resonance > 1.0 (more test code than logic code in PRs). A dropping resonance score is a leading indicator of an architectural "Root Canal"—where logic complexity is outstripping its verification layer.

## 4. The Invisible Glue (Connective Tissue)
*Metric: Author mobility and cross-repo review density.*

**Sight Beyond Sight**: Reveals the "Glue People" who prevent organizational fracturing. These individuals bridge silos by reviewing code in repositories they don't author in, ensuring knowledge diffusion.

## 5. Process Leakage (Shadow Merges)
*Metric: Merged PRs with zero reviews or CI failures.*

**Sight Beyond Sight**: Exposes where "The Rules" are being bypassed to maintain velocity. This is the primary source of architectural "Leaks" that eventually become production cavities.
